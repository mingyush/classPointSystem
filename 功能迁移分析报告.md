## 五、核心业务逻辑分析

### 5.1 积分计算逻辑

```
积分余额 = SUM(所有积分记录.points)
- 扣减操作: points为负数
- 兑换操作: type='purchase'时同时创建订单和积分扣减
- 退款操作: type='refund'时积分返还
```

### 5.2 排行榜计算

```
totalRanking: 按SUM(points)降序排列
dailyRanking: 只统计当日积分变动
weeklyRanking: 只统计本周(周一至今)积分变动
```

### 5.3 订单状态流转

```
pending -> confirmed (教师确认)
pending -> cancelled (学生/教师取消)
confirmed -> cancelled (不允许，实际只取消未确认的)
```

### 5.4 班级互动状态流转

```
notice/task:
  pending -> confirmed (班级确认) -> closed (教师关闭)
  pending -> closed (教师直接关闭)

report:
  pending -> reviewed (教师审核通过/拒绝) -> closed
```

### 5.5 学期激活流程

```
1. 获取当前学期的所有学生积分快照
2. 为每个学生创建 SemesterArchive 记录
3. 清空当前学期的积分记录
4. 将新学期设置为isCurrent=true
5. 可选: 发放新学期初始积分
```
