# 班级积分管理系统 V2 - 功能需求规格说明书

## 一、系统概述

### 1.1 系统定位

面向中学班级的积分管理工具，支持多学期管理、实时展示、数据统计，帮助教师量化学生表现，激励学生进步。

### 1.2 用户角色

| 角色 | 权限范围 |
|------|---------|
| **管理员** | 全部功能 + 系统配置 + 教师管理 + 学期管理 |
| **教师** | 积分操作 + 商品管理 + 订单处理 + 班级管理 |
| **学生** | 查看个人积分 + 预约商品 + 查看排行榜 |

### 1.3 系统目标

- **简化操作**: 教师可快速完成日常积分操作
- **数据可视化**: 大屏实时展示，激励学生
- **可追溯**: 完整的操作记录，支持审计
- **易扩展**: 模块化设计，便于功能扩展

| **阈值模式** | 超过阈值的积分归零，其余保留 | threshold: number |

## 三、非功能性需求

### 3.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面加载时间 | < 2秒 | 首屏加载 |
| API响应时间 | < 500ms | P95延迟 |
| 排行榜计算 | < 1秒 | 含缓存策略 |
| 并发用户 | 50+ | 同时在线 |
| 数据量支持 | 10万+ | 单学期积分记录 |
| 数据库查询 | < 100ms | 常规查询 |

### 3.2 安全要求

| 要求 | 措施 |
|------|------|
| 密码存储 | bcrypt加密，cost≥12 |
| 会话管理 | JWT + HttpOnly Cookie |
| API安全 | CORS、CSRF防护、Rate Limiting |
| 敏感操作 | 二次确认/密码验证 |
| 日志脱敏 | 不记录密码、Token等 |
| SQL注入 | Prisma参数化查询 |
| XSS防护 | 输入验证、输出转义 |

## 四、数据迁移需求

### 4.1 迁移范围

| 数据 | 来源 | 目标 |
|------|------|------|
| 学生信息 | students.json | Student表 |
| 教师信息 | teachers.json | Teacher表 |
| 商品信息 | products.json | Product表 |
| 积分记录 | points.json | PointRecord表 |
| 订单记录 | orders.json | Order表 |
| 系统配置 | config.json | SystemConfig表 |

### 4.2 迁移流程

```
1. 环境准备
   ├── 创建SQLite数据库
   ├── 执行Prisma迁移
   └── 验证表结构

2. 基础数据导入
   ├── 导入学生信息
   ├── 导入教师信息
   └── 导入商品信息

3. 学期数据处理
   ├── 分析积分记录时间范围
   ├── 创建对应学期
   └── 设置学期日期

4. 业务数据导入
   ├── 导入积分记录(关联学期)
   ├── 导入订单记录(关联学期)
   └── 计算学生积分余额

5. 数据验证
   ├── 积分余额校验
   ├── 关联完整性检查
   └── 生成验证报告

6. 收尾工作
   ├── 创建当前学期
   ├── 设置系统配置
   └── 生成迁移报告
```

### 4.3 数据验证规则

| 验证项 | 规则 |
|------|------|
| 学生余额 | balance = Σ(pointRecord.points) |
| 外键关联 | 所有外键必须有效 |
| 数据完整 | 必填字段不能为空 |
| 唯一性 | 学号、教师ID等唯一 |
| 时间逻辑 | 开始时间 < 结束时间 |
